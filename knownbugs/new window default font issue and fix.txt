The fix is in place. Here's a summary of what was changed and why:

**Root cause:** The font block in `_apply_settings_dict` had overly strict guards — it only applied a font if both `font_family` was a non-empty string AND `font_size` was an `int` in range. Two failure modes:
1. If `font_family` was saved as `""` (e.g., when the Nirmala UI fallback `QFont()` had no family name), the condition `if family and ...` would be `False` and no font was applied — leaving the new window with Qt's default font
2. If `font_size` was stored as a float like `12.0` (from an older version), `isinstance(12.0, int)` is `False` in Python

**Fix:**
- In `_apply_settings_dict`: when `font_family` is absent/empty, substitute a platform default ("Segoe UI" on Windows, ".AppleSystemUIFont" on macOS). When `font_size` is a whole-number float, coerce to int. When `font_size` is missing/out of range, default to 12pt. Font is now always applied.
- In `_save_preferences`: when `font.family()` returns an empty string, save a platform default instead of an empty string, so the settings file always has a usable font family.
